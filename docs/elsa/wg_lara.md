# Working Group Licensing & Agreement Rule Automation (LARA)

The Working Group LARA is the newest working group within ELSA. It focuses on tools that help researchers choose an appropriate license for their outputs (data, code, etc.) and automate the creation of data use agreements.

The WG provides a forum for legal and technical discussions on these topics and explores possibilities for further development and integration of existing tools — for example, embedding ODRL metadata for software or enabling the creation of custom licenses from building blocks. The group actively exchanges knowledge with other groups working on related topics to minimize duplicated effort across disciplines.

## Tools

### LicenseChecker

A web-based tool that supports researchers in selecting an appropriate open-source license for their research software or code. It provides an overview of common open-source licenses and allows uploading code and dependency files to check for existing licenses and suggest compatible ones.

- GitHub: [https://github.com/izus-fokus/LicenseChecker](https://github.com/izus-fokus/LicenseChecker)
- Web application: [https://checklicense.uni-stuttgart.de/](https://checklicense.uni-stuttgart.de/)

### ODRL Builder

A web application for creating Data Use Agreements (DUA) that enables automated assessment of access permissions. It adheres to the ODRL (Open Digital Rights Language) standard and models permissions and prohibitions for the use of digital objects.

- GitHub: [https://github.com/IPK-BIT/odrl-builder](https://github.com/IPK-BIT/odrl-builder)
- Web application: [https://odrl.gi.denbi.de/](https://odrl.gi.denbi.de/)
